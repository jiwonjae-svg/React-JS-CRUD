# React CRUD 게시판 (Full-Featured Community)

React JS로 구현한 완전한 기능을 갖춘 테마별 커뮤니티 게시판 애플리케이션입니다.

## 🌟 주요 기능

### 1. 인증 시스템
- ✅ **회원가입**: 사용자명, 이메일, 비밀번호 입력
- ✅ **로그인**: 사용자명 또는 이메일로 로그인
- ✅ **계정 찾기**: 이메일로 사용자명 찾기
- ✅ **비밀번호 재설정**: 이메일 인증을 통한 비밀번호 변경
- 🔐 **회원정보 수정**: 사용자명, 이메일, 비밀번호 변경 기능
- 👁️ **게스트 모드**: 로그아웃 상태에서도 게시글 조회 가능

### 2. 게시글 CRUD
- ✅ **작성(Create)**: 제목, 내용, 카테고리, 미디어 첨부
- ✅ **조회(Read)**: 게시글 목록 및 상세 보기, 조회수 자동 증가
- ✅ **수정(Update)**: 작성자 본인만 게시글 수정 가능
- ✅ **삭제(Delete)**: 작성자 본인만 게시글 삭제 가능 (상세보기에서만)

### 3. 테마별 게시판
- 🎨 **그림 게시판**: 그림/일러스트 관련 게시글
- 🎮 **게임 게시판**: 게임 관련 게시글
- 🎬 **영화 게시판**: 영화/드라마 관련 게시글
- 📚 **만화 게시판**: 만화/웹툰 관련 게시글
- ⚽ **스포츠 게시판**: 스포츠 관련 게시글
- 📋 **전체 게시판**: 모든 카테고리 통합 보기

### 4. 댓글 시스템
- 💬 댓글 작성 및 조회
- 🗑️ 댓글 삭제 (작성자만)
- 📊 댓글 개수 표시

### 5. 미디어 첨부
- 📷 이미지 파일 업로드 및 미리보기
- 🎥 비디오 파일 업로드 및 재생
- 🖼️ 여러 개의 미디어 파일 첨부 가능
- ✕ 첨부 파일 제거 기능

### 6. 좋아요 시스템
- ❤️ 게시글 좋아요/취소
- 👤 사용자별 좋아요 중복 방지
- 📈 좋아요 개수 실시간 업데이트

### 7. 인기 게시글
- 🔥 좋아요 기준 일일 인기 게시글 표시
- 📊 좌측 사이드바에 TOP 5 게시글
- 📱 반응형 디자인 (모바일에서는 숨김)

## 🚀 설치 및 실행

### 1. 의존성 설치
```bash
npm install
```

### 2. 개발 서버 실행
```bash
npm run dev
```

### 3. 브라우저에서 확인
- 터미널에 표시되는 URL(보통 http://localhost:5173)로 접속

### 4. PowerShell 실행 정책 오류 발생 시
```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

## 🛠️ 사용 기술

- **React 18**: UI 라이브러리
- **Vite**: 빌드 도구
- **Context API**: 전역 상태 관리
- **LocalStorage**: 데이터 영속성
- **CSS3**: 스타일링 (Flexbox, Grid, Gradient)

## 📖 사용 방법

### 게스트 모드 (로그아웃 상태)
- 게시글 목록 조회
- 게시글 상세보기
- 카테고리별 필터링
- 인기 게시글 확인

### 회원 모드

#### 회원가입 & 로그인
1. **회원가입**: 로그인 화면에서 '회원가입' 클릭
2. **정보 입력**: 사용자명, 이메일, 비밀번호 입력
3. **로그인**: 가입한 정보로 로그인

#### 회원정보 관리
1. 로그인 후 헤더의 '회원정보' 버튼 클릭
2. 사용자명, 이메일 수정 가능
3. 비밀번호 변경 (현재 비밀번호 확인 필요)

#### 게시글 관리
1. **카테고리 선택**: 상단의 카테고리 버튼으로 테마 선택
2. **작성**: '글쓰기' 버튼 → 카테고리 선택 → 내용 입력 → 미디어 첨부 (선택) → '작성'
3. **조회**: 제목 클릭하여 상세보기
4. **수정**: 본인 게시글 상세보기 → '수정' 버튼
5. **삭제**: 본인 게시글 상세보기 → '삭제' 버튼

#### 댓글 & 좋아요
1. 게시글 상세보기로 이동
2. 하단 댓글 입력란에 내용 작성
3. '댓글 작성' 버튼 클릭
4. 좋아요 버튼(하트) 클릭으로 좋아요/취소

## ✨ 주요 특징

- 📱 **반응형 디자인**: 모든 화면 크기에 최적화
- 🎯 **직관적인 UX**: 간단하고 명확한 사용자 인터페이스
- ⚡ **빠른 성능**: Vite를 사용한 빠른 개발 환경
- 💾 **데이터 영속성**: LocalStorage를 통한 데이터 저장
- 🔐 **보안**: 권한 기반 접근 제어
- 🎨 **모던한 UI**: 그라디언트와 부드러운 애니메이션
- 👁️ **게스트 친화적**: 로그인 없이도 컨텐츠 조회 가능
- 🏷️ **테마 분류**: 카테고리별 게시글 필터링

## 🎨 UI 개선 사항

- 인기 게시글 사이드바 폰트 크기 축소 (13px)
- 게시글 목록 폰트 크기 조정 (14px)
- 긴 제목 자동 말줄임(...) 처리
- 게시글 목록에서 '관리' 탭 제거 (삭제는 상세보기에서만)
- 컴팩트한 인기글 리스트 디자인

## 💡 추가 기능 가능성

- 게시글 검색 기능
- 페이지네이션
- 해시태그 시스템
- 알림 시스템
- 다크 모드
- 댓글에 대댓글 기능

## 🐛 알려진 이슈

- 미디어 파일은 Base64로 인코딩되어 LocalStorage에 저장되므로, 큰 파일(5MB 이상) 업로드 시 용량 제한이 있을 수 있습니다.
- 실제 서비스 환경에서는 백엔드 API와 파일 서버가 필요합니다.
- LocalStorage는 브라우저당 약 5-10MB 제한이 있습니다.

## 🔒 보안 참고사항

- 현재 비밀번호는 평문으로 저장됩니다 (실제 서비스에서는 해시화 필요)
- LocalStorage는 클라이언트 측 저장소로 보안에 취약할 수 있습니다
- 실제 프로덕션에서는 JWT 토큰과 백엔드 API 사용을 권장합니다
